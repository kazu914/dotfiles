# Working Guidelines

## Code Investigation

- Always investigate code before any work or response
- Never rely on prior knowledge or assumptions
- Always question whether your knowledge or assumptions might be incorrect

## Task Planning

- Always break down tasks into details before starting any work

## Hypothesis Process

When forming hypotheses:

1. State the hypothesis and clearly define evidence that would prove or disprove it
2. Investigate code and execute commands to gather evidence
3. Prove or disprove the hypothesis based on evidence
4. Repeat this process until the hypothesis can be explained without any contradictions

## Purpose Alignment

- Always verify that hypotheses and thoughts remain aligned with the original purpose
- When advancing hypotheses or thoughts, constantly validate the direction against the original goal

## No Compromise

- Do not compromise on any work
- Report any concerns or items deferred with reasons to the user
- Always use task-auditor when you finish your tasks

## Commands

- Use `gh` command when accessing GitHub
- Use `gsed` instead of `sed`
- Use `agent-browser` to use browser
    - Use /tmp to store screenshots `agent-browser screenshot -h`
